Latin Square Experiment

Latin Square are turned into a COP as we try to maximize the sum of the diagonal.

Different models and branching schemes are compared on the same instances.

  • sum_product-max_marginal_regret: bp with sum product, branching with max marginal regret
  • sum_product-max_marginal_regret-oracle: bp with sum product, oracle on objective, branching with max marginal regret
  • max_product-max_marginal_regret-oracle: bp with max product, oracle on objective, branching with max marginal regret
  • no_bp-first_fail: select variable with smallest domain, select smallest value
  • no_bp-max_value: select the max value on the diagonal

max_value generally the fastest by far. Is beaten once by max_product though.

max_product and max_value find the only one solution, the optimal one.

Code 
import plot

instance_numbers = range(1, 11)

models = [
  'dfs-sum_product-max_marginal_regret',
  'dfs-sum_product-max_marginal_regret-oracle',
  'dfs-max_product-max_marginal_regret-oracle',
  'dfs-no_bp-first_fail',
  'dfs-no_bp-max_value',
]

Impact of a constraint z >= 0.5 * max(z)

Truncating the domain of the objective to 50% of its maximum value. Won’t make instances unsat or even skip first solutions found (they are usually above this threshold).

Mostly it doubles the slope of the oracle function.

max_product-max_marginal_regret-oracle is almost unaffected.

Various effects for the other models : speed up, slow down, no effect… depending on the instance.

Slow down is really surprising for sum_product-max_marginal_regret (no-oracle) as it supposedly only restricts the search space.

Graphs : one color for each model, lighter and left is untruncated, darker and right is truncated.
